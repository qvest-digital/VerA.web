calendar-setup.js

This file provides an easy way to setup the calendar. It defines a helper function that does all the needed stuff, including creating select and close handlers and patching the input field or the button to open a popup calendar when clicked. It can create a single-click or double-click popup calendar. It can work with a hidden field, in which case you are advised ;-) to display the current date in another element (also supported through this function).

Note that this file is only useful for creating popup calendars. For creating a flat calendar you still need to go with the "classic" method, presented in the main demo file.

Function prototype

Calendar.setup( parameters );

Here is an example of how could you call it. For more demos visit simple-1.html and check its source.

<input type="text" id="data" name="data" /><button id="trigger">...</button>

<script type="text/javascript">
  Calendar.setup(
    {
      inputField  : "data",    // ID of the input field
      ifFormat    : "M d, y",  // the date format
      button      : "trigger"  // ID of the button that triggers the calendar
    }
  );
</script>

Following there is the complete list of properties. Note that they are not directly parameters to the function. Instead, the function receives only one parameter—a JavaScript object. The properties are then members of this object. I used this because it looks much better than simply passing comma-separated parameters that have no name. As you might notice ;-) the idea comes from Perl.

property description default value
inputField [ string ] The ID of your input field. null
displayArea [ string ] This is the ID of a <span>, <div>, or any other element that you would like to use to display the current date. This is generally useful only if the input field is hidden, as an area to display the date. null
button [ string ] The ID of the calendar "trigger". This is an element (ordinarily a button or an image) that will dispatch a certain event (usually "click") to the function that creates and displays the calendar. null
eventName [ string ] The name of the event that will trigger the calendar. The name should be without the "on" prefix, such as "click" instead of "onclick". Virtually all users will want to let this have the default value ("click"). Anyway, it could be useful if, say, you want the calendar to appear when the input field is focused and have no trigger button (in this case use "focus" as the event name). "click"
ifFormat [ string ] The format string that will be used to enter the date in the input field. This format will be honored even if the input field is hidden. "y/mm/dd"
daFormat [ string ] Format of the date displayed in the displayArea (if specified). "y/mm/dd"
singleClick [ boolean ] Wether the calendar is in "single-click mode" or "double-click mode". If true (the default) the calendar will be created in single-click mode. true
mondayFirst [ boolean ] If "true" then the calendar will display with Monday being the first day of week. false
align [ string ] Alignment of the calendar, relative to the reference element. The reference element is dynamically chosen like this: if a displayArea is specified then it will be the reference element. Otherwise, the input field is the reference element.
For the meaning of the alignment characters please see the calendar documentation. 		"Bl"
range [ array ] An array having exactly 2 elements, integers. (!) The first [0] element is the minimum year that is available, and the second [1] element is the maximum year that the calendar will allow. [1900, 2999]
